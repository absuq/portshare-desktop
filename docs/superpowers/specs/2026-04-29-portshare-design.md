# 端口发布器设计规格

## 背景

目标是做一个桌面端端口发布工具，用来把本机 HTTP/HTTPS 服务灵活开放给另一台电脑，默认面向 Tailscale tailnet 内访问，并允许用户手动开启公网访问。工具首版以 Tailscale 为发布后端，但架构不能绑定 Tailscale，后续应能接入自建中转或其他发布 provider。

## 已确认目标

- 使用 Go 开发。
- 使用 Fyne 做桌面图形界面。
- 默认中文界面，支持切换英文。
- 专有词保留原文，例如 Tailscale、Serve、Funnel、MagicDNS、tailnet、provider。
- 默认只开放到 tailnet。
- 公网发布必须手动触发并强确认。
- 公网发布支持倒计时关闭，也支持明确选择长期开放。
- 支持多个端口同时开放和管理。
- 首版只支持 HTTP/HTTPS 网页服务。
- 首版支持自动发现本地服务和手动添加服务。
- 支持系统托盘常驻。
- 保存历史记录，并支持日志保留时间，默认保留 1 年。
- 首版以 Tailscale 为 provider，但核心应用按 provider 抽象设计。
- 需求沟通结束后维护 AGENTS.md，记录已完成、计划完成和未来需完成 todo。

## 非目标

首版不实现原始 TCP 发布，不实现公网访问令牌、密码页或认证代理，不实现自建中转服务端，不做 provider 插件市场，不提供复杂多用户权限模型。

## 架构设计

应用分为界面层、应用层、provider 层、发现层、日志层和配置层。

界面层负责 Fyne 窗口、中文/英文界面、服务列表、确认弹窗、托盘菜单和状态展示。窗口关闭时默认最小化到托盘，不直接停止发布。

应用层负责添加服务、开启 tailnet 发布、开启公网发布、倒计时关闭、长期开放确认、停止发布、刷新状态和调度日志写入。计时关闭逻辑放在应用层，不放在界面层。

Provider 层定义统一发布接口。上层只使用本地服务、发布模式、访问地址、状态、关闭动作和 provider 能力，不直接依赖 Tailscale 命令概念。首版实现 TailscaleProvider，后续可实现 CustomRelayProvider、FRPProvider 或其他自建中转 provider。

发现层扫描本机监听端口，并对可能的 HTTP/HTTPS 服务做轻量探测。能识别页面标题时显示标题，不能识别时显示为本地服务加端口号。

日志层记录开放、关闭、失败、到期关闭、程序退出等事件。日志默认保留 1 年，可配置为 30 天、90 天、1 年或永久。

配置层保存语言、日志保留时间、端口列表、窗口行为、托盘行为、长期开放偏好和 provider 相关配置。

## Provider 抽象

Provider 接口需要描述能力，而不是具体实现细节：

- 是否支持 tailnet 发布
- 是否支持公网发布
- 是否支持过期时间
- 是否支持多端口
- 是否支持自定义域名
- 是否支持状态查询
- 是否支持关闭指定发布

核心行为包括：

- 发布到 tailnet
- 发布到公网
- 停止单个发布
- 停止全部发布
- 查询当前发布状态
- 查询 provider 健康状态和能力

首版 TailscaleProvider 可使用 Tailscale 本地能力实现。理想方向是优先使用本地 API；如果 Serve/Funnel 的本地 API 能力不足或权限复杂，可在 TailscaleProvider 内部使用 CLI fallback，但这种细节不能泄漏到应用层。

## 界面设计

主界面是端口列表管理器。每个服务项显示本地地址、服务名称、tailnet 状态、公网状态、访问地址、剩余开放时间和操作按钮。

主操作包括：

- 刷新发现
- 手动添加服务
- 开放到 tailnet
- 开启公网
- 复制访问地址
- 关闭单个发布
- 停止全部发布

公网开启流程必须醒目。用户点击开启公网后，先出现强确认，再选择开放时长。常用时长包括 10 分钟、30 分钟、1 小时、自定义时间和长期开放。选择长期开放时必须二次确认。公网开放期间，主界面和托盘都持续显示公网状态，并始终提供一键关闭入口。

设置页包含语言、日志保留时间、关闭窗口行为、退出确认、provider 状态和未来 provider 配置入口。默认语言为中文，英文可切换，重启后保留选择。

## 数据流和行为

应用启动后加载配置、语言和日志保留策略，然后检查 provider 可用性和能力。随后发现层扫描本地监听端口，生成候选服务列表。用户可以从候选服务开放，也可以手动添加服务。

tailnet 发布流程：

1. 用户选择本地服务。
2. 应用层调用 provider 发布到 tailnet。
3. provider 返回 MagicDNS/IP 访问地址。
4. 应用更新服务状态。
5. 日志层记录发布事件。

公网发布流程：

1. 用户选择本地服务。
2. 应用弹出强确认。
3. 用户选择开放时长或长期开放。
4. 应用层调用 provider 发布公网地址。
5. 应用显示公网 URL 和风险状态。
6. 如果是限时开放，应用层启动倒计时任务。
7. 日志层记录公网发布事件。

倒计时到期后，应用层调用 provider 停止公网发布，刷新状态，并写入到期关闭日志。长期开放不自动关闭，但持续显示风险状态。

托盘行为：

- 关闭窗口时最小化到托盘。
- 托盘菜单提供打开主界面、暂停所有公网、停止全部发布和退出程序。
- 有公网开放时退出程序必须确认。
- 退出确认提供退出并关闭所有发布的路径。

状态刷新包括用户手动刷新和后台周期刷新。后台刷新用于发现外部状态变化，例如用户在命令行执行了 Tailscale reset。发现状态不一致时，应用提示重新同步。

## 错误处理和安全策略

本地服务不可用时，界面显示本地服务不可访问，并允许重新检测或编辑地址。常见原因包括端口未监听、HTTP 探测失败和 HTTPS 自签名证书。

Provider 不可用时，界面显示具体原因和建议操作，例如 Tailscale 未运行、未登录、版本不支持 Serve/Funnel 或 Funnel 权限未启用。

发布状态不一致时，应用标记状态已变化，并提示用户刷新或重新同步。

公网安全策略：

- 默认只开放到 tailnet。
- 公网发布必须手动触发。
- 自动发现不能自动开启公网。
- 公网发布必须强确认。
- 长期开放必须二次确认。
- 有公网开放时，主界面和托盘必须持续提示。
- 有公网开放时，退出程序必须提醒。
- 首版不做认证代理，只做风险提示和状态管理。
- 复制或打开公网链接时，界面提示该链接是公网入口。

未来安全增强包括公网访问令牌、简单密码页、OAuth、身份头透传、每个服务的访问策略和原始 TCP 风险分级。

## 测试和验收

服务发现验收：

- 启动 127.0.0.1:3000 测试网页服务，应用能发现并显示。
- 手动添加 localhost:5173，应用能保存并展示。
- HTTPS 自签名服务能被识别为 HTTPS，并标记证书提示。

tailnet 发布验收：

- 选择服务后开放到 tailnet，返回 MagicDNS/IP 访问地址。
- 另一台电脑能通过浏览器访问。
- 关闭发布后另一台电脑不能再访问。

公网发布验收：

- 开启公网时必须出现强确认。
- 选择限时开放后显示倒计时。
- 到期后自动关闭公网发布，并写入日志。
- 长期开放必须二次确认，且主界面和托盘持续提示。

多端口验收：

- 同时开放 3000、5173、8080。
- 单独关闭其中一个不影响其他端口。
- 停止全部发布能关闭所有发布。

托盘验收：

- 关闭窗口后程序仍在托盘。
- 托盘菜单能打开窗口、暂停所有公网、停止全部发布和退出。
- 有公网开放时退出必须提醒。

日志和配置验收：

- 开放、关闭、到期关闭和异常失败都写入历史。
- 日志保留策略默认 1 年。
- 语言默认中文。
- 切换英文后界面文案变化，重启后保留。

状态同步验收：

- 外部重置发布状态后，应用能检测到状态变化并提示同步。

测试分三类：

- 单元测试覆盖 provider 接口、计时关闭、日志保留和配置读写。
- 集成测试使用假的 provider 模拟发布成功、失败和状态变化。
- 手动验收使用真实 Tailscale Serve/Funnel 和两台电脑进行访问测试。

## 后续计划

首版完成后，优先补充原始 TCP 发布、访问令牌或认证代理、自建中转 provider、更多 provider 后端、provider 配置界面和更完整的发布策略。
